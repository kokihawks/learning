# ロードバランサー 問題集

---

## 問題1

ある企業が、マイクロサービスアーキテクチャを構築しています。URLパスに基づいて異なるターゲットグループにルーティングする必要があります。また、WebSocket接続もサポートする必要があります。

この要件を満たすために、最も適切なロードバランサーはどれですか？

A. Classic Load Balancer（CLB）

B. Network Load Balancer（NLB）

C. Application Load Balancer（ALB）

D. Gateway Load Balancer（GWLB）

---

<details>
<summary>解答1を表示</summary>

### 正解

**正解: C**

### 解説

**A. 誤り** - Classic Load Balancerはレガシーサービスであり、新規の使用は推奨されません。また、URLパスに基づくルーティングの機能が制限されています。

**B. 誤り** - Network Load Balancerはレイヤー4（TCP/UDP）で動作するため、URLパスに基づくルーティングはできません。コンテンツベースのルーティングには対応していません。

**C. 正しい** - Application Load Balancerはレイヤー7（HTTP/HTTPS）で動作し、URLパス、ホストヘッダー、HTTPメソッドなどに基づくコンテンツベースのルーティングが可能です。また、WebSocket接続もサポートしています。

**D. 誤り** - Gateway Load Balancerは、サードパーティの仮想アプライアンス（ファイアウォール、侵入検知システムなど）を透過的に統合するためのロードバランサーであり、この要件には適していません。
