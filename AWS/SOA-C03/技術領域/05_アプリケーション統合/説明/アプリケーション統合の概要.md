# AWS アプリケーション統合の概要

## はじめに

AWSのアプリケーション統合サービスは、マイクロサービスアーキテクチャや分散システムにおいて、サービス間の通信と連携を実現するためのサービスを提供します。

## アプリケーション統合の主要なトピック

### 1. SQS（Simple Queue Service）
- SQSキュー（Standard、FIFO）
- メッセージの送受信
- デッドレターキュー（DLQ）
- 可視性タイムアウト
- ロングポーリングとショートポーリング

### 2. SNS（Simple Notification Service）
- SNSトピック
- サブスクリプション（メール、SMS、HTTP/HTTPS、SQS、Lambda）
- フィルターポリシー
- メッセージ属性

### 3. EventBridge（CloudWatch Events）
- イベントバス
- ルールとターゲット
- イベントパターン
- スケジュールルール
- カスタムイベントバス

### 4. API Gateway
- REST APIとHTTP API
- リソースとメソッド
- 統合（Lambda、HTTP、AWSサービス）
- 認証と認可
- レート制限とスロットリング

### 5. Step Functions
- ステートマシン
- ステートタイプ（Task、Choice、Wait、Parallel、Mapなど）
- エラーハンドリング
- 実行履歴

### 6. その他の統合サービス
- AppSync（GraphQL）
- MQ（メッセージブローカー）
- EventBridge Pipes
